# AGENTS.md — Coding Agent Guide

This file describes the codebase structure, conventions, and workflows for AI coding agents (Codex, Claude Code, Cursor, etc.).

## Repository Layout

```
wubi86-trainer/
├── rime-source/              # Raw RIME dict files (not committed, see Setup)
├── app/                      # Vite + React app (all source code lives here)
│   ├── scripts/
│   │   └── build-dict.mjs   # Dict generation script (Node ESM, no deps)
│   ├── src/
│   │   ├── data/            # Generated TS data files + static data
│   │   ├── pages/           # One file per page/view
│   │   ├── store/           # Redux slice + store config + typed hooks
│   │   ├── index.css        # Tailwind v4 entry (@import "tailwindcss")
│   │   └── main.tsx         # App root + page routing (simple useState)
│   ├── index.html
│   ├── vite.config.ts
│   └── tsconfig.json
├── README.md
├── AGENTS.md                 # This file
└── LICENSE
```

## Setup (first time)

```bash
# 1. Get the raw dict (not in git)
git clone --depth=1 https://github.com/KyleBing/rime-wubi86-jidian.git rime-source

# 2. Install deps
cd app && pnpm install

# 3. Generate TS data files from the raw dict
pnpm build:dict
```

**Never edit `src/data/*.ts` by hand** — they are fully generated by `scripts/build-dict.mjs`.

## Key Conventions

### Routing

There is no react-router. Routing is a `useState<Page>` in `main.tsx`. Pages are rendered conditionally (`{page === 'x' && <XPage />}`). To add a page:

1. Create `src/pages/MyPage.tsx`
2. Add `'mypage'` to the `Page` type in `main.tsx`
3. Add a nav button and a conditional render

### State Management

All persistent state lives in `src/store/progressSlice.ts`:

- `lessons`: one `LessonProgress` per lesson key (`level1 | level2 | phrase | article`)
- `articleSession`: ephemeral timing/WPM data for article practice

Always use `useAppDispatch` / `useAppSelector` from `src/store/hooks.ts` — never import raw `useDispatch`/`useSelector`.

Available actions: `advance`, `recordMistake`, `resetLesson`, `jumpTo`, `startArticle`, `articleCorrect`, `articleMistake`, `finishArticle`, `resetArticleSession`.

### Data Files

| File | Purpose |
|------|---------|
| `level1.ts` | 25 level-1 chars (`LEVEL1: Level1Entry[]`) |
| `level2.ts` | 616 level-2 chars (`LEVEL2: Level2Entry[]`) |
| `level2Groups.ts` | level-2 grouped by first key (`LEVEL2_GROUPS`) |
| `phrases.ts` | Word pairs: `PHRASES_TIER1` (≥30 weight), `PHRASES_TIER2` (=20) |
| `dict.ts` | Full single-char lookup `DICT: Map<string, string[]>` |
| `codeUtils.ts` | `lookupCode(char)`, `parseArticle(text)`, `isCJK(char)` |
| `articles.ts` | Built-in practice articles (`ARTICLES: Article[]`) |
| `keyboard.ts` | Keyboard row layout constants |

### Styling

Tailwind CSS v4 — imported via `@import "tailwindcss"` in `index.css`. No config file needed. Custom keyframes (e.g. `shake`) are defined directly in `index.css`.

Use `bg-gray-950` for the page background. Color palette per mode:
- Level 1: `amber-400`
- Level 2: `sky-400`
- Phrase: `violet-400`
- Article: `emerald-400`

### TypeScript Strictness

`tsconfig.json` has `"strict": true` and `"noUnusedLocals": true`. Type imports must use `import type { ... }` syntax. Do **not** re-enable `verbatimModuleSyntax` — it was intentionally removed.

## Common Tasks

### Add a new practice article

Edit `src/data/articles.ts`, append to `ARTICLES`:

```ts
{
  id: 'unique-id',
  title: '文章标题',
  difficulty: 1 | 2 | 3,
  description: '简短描述',
  content: `全文内容...`,
}
```

No rebuild needed — articles are static data, not generated.

### Update the dict source

```bash
cd rime-source && git pull
cd ../app && pnpm build:dict
```

### Add a new lesson mode

1. Add a new key to `LessonKey` in `progressSlice.ts` and add a `defaultLesson()` entry in `initialState.lessons`
2. Create the page component in `src/pages/`
3. Wire up routing in `main.tsx`
4. Optionally wire to `ResultPage` for round summaries

### Run type check only (no build)

```bash
cd app && pnpm exec tsc --noEmit
```

## Build

```bash
cd app
pnpm build       # production build → app/dist/
pnpm dev         # dev server at localhost:5173
pnpm build:dict  # regenerate TS data from rime-source
```

Chunk split strategy (see `vite.config.ts`): `vendor-react`, `vendor-redux`, `data-phrases`, `data-dict` are split into separate chunks to keep the main bundle small.

## Do Not

- Do not edit any file under `src/data/` that is listed as auto-generated (check the top comment)
- Do not add `verbatimModuleSyntax` back to tsconfig
- Do not install a router library without discussion — the current `useState` routing is intentional for simplicity
- Do not use `rm` for deletions — prefer safe alternatives
- Do not commit `rime-source/` (it's in `.gitignore`)
